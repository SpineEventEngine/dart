Creates a new ActorRequestFactory.

An actor is the ID of a user who initiates the request. If there is no user ID (e.g. before the login) a conventional user ID should be used. Usually, Anonymous is chosen for the absent user ID.

In multitenant systems, it's required for all the actor requests to have a tenant ID.
